# Express 内存泄露示例

一个简单的 Express 内存泄露演示。

## 内存泄露原因

每个请求都会在全局 `map` 中存储数据，但从不清理，导致内存持续增长。

## 使用方法

```bash
# 安装依赖
npm install

# 启动服务器
npm start
```

## 测试

1. 启动服务器后，不断访问 `http://localhost:3000/`
2. 观察控制台输出，可以看到 Map 大小和内存使用量持续增长
3. 访问 `http://localhost:3000/memory` 查看当前内存状态

## 预期结果

随着请求数量增加，内存使用量会持续增长，这就是内存泄露的表现。
